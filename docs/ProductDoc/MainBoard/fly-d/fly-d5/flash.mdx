# 固件编译与烧录

## 准备工作

    * 开始获取ID前，请先[连接到SSH](/docs/DebugDoc/BasicTutorial/index.mdx "点击即可跳转") 
    * 请确保下位机连接上位机的数据线具有数据传输功能
    * SSH连接连接到上位机后输入`lsusb`并且回车，将可能出现下面几种设备
    * 如果系统不支持`lsusb`可以执行下方指令进行安装
    * 如果`lsusb`后还是没有设备，请更换上位机！

    ```
    sudo apt-get install usbutils
    ```

        <ImageView image={require('@site/docs/General/get-id/img/katapult.png').default} size="100%" align="left" />

    * `1d50:614e`属于Klipper固件是你的下位机刷了正确的USB固件才会显示，如果刷错固件或者未刷固件则不会显示设或者显示其他设备。
    * `1d50:6177`属于Katapult固件是你的下位机刷了正确的USB固件才会显示，如果刷错固件或者未刷固件则不会显示设或者显示其他设备。
    * 输入`ls /dev/serial/by-id/*` 回车。如果一切正常，则会出现一行ID。
    * 如果输入`ls /dev/serial/by-id/*`后没有ID出来，但是`lsusb`有设备请更换上位机或者更换上位机系统

## 开始烧录

:::tip

使用Katapult固件可以方便以后直接通过SSH更新固件，无需在进入DFU烧录固件
:::

### 安装烧录插件

:::tip
如果已经安装过则无需再次安装
:::

    * 安装`pip install pyserial`

    ```bash
    pip install pyserial
    ```

    * 如果提示`-bash: pip: command not found`需要执行

    ```bash
    sudo apt-get install pip
    ```

    * 如果提示出现下方报错需要执行

    <ImageView image={require('@site/docs/General/get-id/img/pyserial.png').default} size="100%" align="left" />

    ```bash
    pip install pyserial --break-system-packages
    ```

### 开始烧录固件

    *烧录固件使用下方代码，请确保已经编译好固件，请注意`=`后面没有空格

    ```bash
    make flash FLASH_DEVICE=<你的设备串口地址>
    ```

### 更新固件方法


    * 更新USB固件需要在编译好最新固件后输入下方命令更新，请注意`=`后面没有空格

    ```bash
    make flash FLASH_DEVICE=<你的设备串口地址>
    ```

    * 更新USB桥接CAN固件需要在编译好最新固件后输入下方命令重置进去Katapult

    ```bash
    ~/klippy-env/bin/python3 ~/katapult/scripts/flashtool.py -i can0 -u <MCU ID> -r
    ```

    * 重置进去Katapult后在执行下方执行更新固件

    ```bash
    make flash FLASH_DEVICE=<你的设备串口地址>
    ```
