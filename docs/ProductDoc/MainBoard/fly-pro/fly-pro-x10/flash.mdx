## 准备工作

    * 开始获取ID前，请先[使用网络连接SSH](/docs/DebugDoc/BasicTutorial/index.mdx "点击即可跳转") 
    * 请确保下位机连接上位机的数据线具有数据传输功能
    * SSH连接连接到上位机后输入`lsusb`并且回车，将可能出现下面几种设备
    * 如果系统不支持`lsusb`可以执行下方指令进行安装
    * 如果`lsusb`后还是没有设备，请更换上位机！

    ```
    sudo apt-get install usbutils
    ```

        <ImageView image={require('@site/docs/General/get-id/img/lsusb.png').default} size="100%" align="left" />

    * `1d50:614e`属于Klipper的USB固件是你的下位机刷了正确的USB固件才会显示，如果刷错固件或者未刷固件则不会显示设或者显示其他设备。
    * `1d50:606f`属于Klipper的USB桥接CAN固件是你的下位机刷了正确的USB固件才会显示，如果刷错固件或者未刷固件则不会显示设或者显示其他设备。
    * `1d50:6177`属于Katapult的USB桥接CAN固件是你的下位机刷了正确的USB固件才会显示，如果刷错固件或者未刷固件则不会显示设或者显示其他设备。
    * `0483:df11`属于STM32 DFU设备可烧录固件
    * 输入`ls /dev/serial/by-id/*` 回车。如果一切正常，则会出现一行ID。
    * 如果输入`ls /dev/serial/by-id/*`后没有ID出来，但是`lsusb`有设备请更换上位机或者更换上位机系统
