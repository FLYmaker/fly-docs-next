# Erweiterung der LED-Steuerung durch Plugins

Originaldokument

```cfg
https://github.com/julianschill/klipper-led_effect
```

## Installation

Verwenden Sie **MobaXterm_Personal** oder ein ähnliches **SSH-Tool**, um eine Verbindung über **WIFI** zu Ihrem Host-Computer herzustellen und stellen Sie sicher, dass folgende Punkte erfüllt sind:

1. **Stellen Sie sicher, dass der Host-Computer die Klipper-Firmware installiert hat**
2. **Stellen Sie sicher, dass Sie sich als der Benutzer anmelden, der Klipper installiert hat**
3. **Stellen Sie sicher, dass Ihr Eingabemodus auf Englisch eingestellt ist**
4. **Stellen Sie sicher, dass Ihr Host-Computer das Gerät normalerweise erkennen kann**
5. **Stellen Sie sicher, dass alle obigen Punkte erfüllt sind, andernfalls können Sie nicht zum nächsten Schritt übergehen**

Dieses Modul kann mithilfe eines Installationsskripts in einer bestehenden Klipper-Installation installiert werden.

```cfg
cd ~
git clone https://github.com/julianschill/klipper-led_effect.git
cd klipper-led_effect
./install-led_effect.sh
```

## Referenzkonfiguration

```cfg
[neopixel TP_led]
pin: PB7
chain_count: 25
# Anzahl der LEDs
# Anzahl der Leuchtdioden
color_order: GRB
initial_RED: 0.4    
initial_GREEN: 0.8
initial_BLUE: 1
initial_WHITE: 0.0
#66CCFF 

[led_effect sb_nozzle_cooling]
autostart:              false
frame_rate:             24
leds:
    neopixel:TP_led (9,10)
layers:
        breathing  3 1 top (0.0, 0.0, 1.0, 0.1)

[led_effect rainbow]
leds:
    neopixel:TP_led
autostart:                          true
frame_rate:                         24
layers:
    gradient  0.3  1 add (0.3, 0.0, 0.0),(0.0, 0.3, 0.0),(0.0, 0.0, 0.3)

```
