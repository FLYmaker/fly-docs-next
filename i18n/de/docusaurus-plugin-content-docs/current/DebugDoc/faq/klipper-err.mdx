# Häufige Fehlermeldungen

## mcu 'xxx': Kann keine Verbindung herstellen

1. **Fehlermeldung**

   <ImageView image={require('@site/docs/DebugDoc/faq/img/cannot_connect.png').default} size="80%" align="center" />

2. **Fehlerursache**

   * Der Host kann das Hauptboard nicht finden und eine Verbindung herstellen

3. **Lösungsansätze**

   * Überprüfen Sie die Klippy.log und scrollen Sie ganz nach unten
   * Erscheint `[Errno 2]`, bedeutet dies, dass keine ID gesucht und in **printer.cfg** hinzugefügt wurde

   <ImageView image={require('@site/docs/DebugDoc/faq/img/errno2.png').default} size="80%" align="center" />

   * Erscheint **serialhdl.error: mcu 'xxx': Serial connection closed**, bedeutet dies, dass die CAN ID nicht gefunden oder falsch eingegeben wurde. Es ist notwendig, die ID zu suchen und einzugeben

   <ImageView image={require('@site/docs/DebugDoc/faq/img/lnvalid_can_uuid.png').default} size="80%" align="center" />
 
   * Erscheint **mcu 'mcu': Unable to open CAN port: [Errno 19] No such device**, bedeutet dies, dass kein **UOTC oder ein Hauptboard mit CAN-Brücke** vorhanden ist

   <ImageView image={require('@site/docs/DebugDoc/faq/img/errno19.png').default} size="80%" align="center" />

   * Erscheint **mcu 'mcu': Unable to open CAN port: Failed to transmit: [Errno 100] Network is down**, bedeutet dies, dass das **CAN-Netzwerk nicht konfiguriert oder CAN0 nicht gestartet ist**

   * Erscheint **mcu 'mcu': Unable to open CAN port: Failed to transmit: [Errno 105] Network is down**, bedeutet dies, dass der **CAN-Netzwerkspeicher zu klein ist**. Lösung: Den Speicher vergrößern

   * Erscheint `[Errno 105] Network is down` oder `[Errno 100] Network is down`, schauen Sie sich dieses Dokument an: [CAN-Konfiguration](/docs/DebugDoc/faq/klipper_can.mdx "Klicken Sie hier, um dorthin zu springen")

## Fehler 'XXX 'mcu' shutdown: ADC out of range….

1. **Fehlermeldung**

   <ImageView image={require('@site/docs/DebugDoc/faq/img/adc.png').default} size="80%" align="center" />

2. **Fehlerursache**

   * Klipper erkennt, dass die Temperatur den maximalen oder minimalen Schwellenwert überschreitet (Thermistor könnte unterbrochen oder kurzgeschlossen sein), das System geht in den Abschaltmodus.

3. **Lösungsansätze**

   * Suchen Sie in der Konfiguration nach `min_temp:` und ändern Sie die Mindesttemperatur auf -235
   * Suchen Sie in der Konfiguration nach `max_temp:` und ändern Sie die Maximaltemperatur auf 99999999
   * Überprüfen Sie, ob der Thermistor-Typ korrekt ist
   * Überprüfen Sie, ob die Pin-Belegung des Thermistors korrekt ist
   * Überprüfen Sie, ob der Thermistor kurzgeschlossen ist
   * Nach Bestimmung und Lösung des Problems müssen die Temperaturen wieder auf den ursprünglichen Wert geändert werden

## MCU 'mcu' shutdown: Timer too close

1. **Fehlermeldung**

   <ImageView image={require('@site/docs/DebugDoc/faq/img/ttc.png').default} size="80%" align="center" />

2. **Fehlerursache**

   * Hauptursache ist die Überlastung der unteren Maschine oder ein Timeout der oberen Maschine

3. **Lösungsansätze**

   * Reduzieren Sie die Mikrosteuerung
   * Verringern Sie die Druckgeschwindigkeit
   * Flashen Sie das Betriebssystem der oberen Maschine neu und flashen Sie die Firmware neu

### Dieses Verfahren kann auf dem Raspberry Pi verwendet werden

   ```bash
   sudo timedatectl status | grep "NTP service" 
   sudo timedatectl set-ntp false
   ```

   * Überprüfen Sie den Status des NTP-Dienstes
   * Deaktivieren Sie die NTP-Synchronisation

## Kommunikationszeitüberschreitung während des Homing-Prozesses

1. **Fehlermeldung**

   Keine Abbildung verfügbar

2. **Fehlerursache**

   * Gleichzeitige CAN- und Bildschirmdienste haben eine hohe Wahrscheinlichkeit, diesen Fehler zu verursachen

3. **Lösungsansätze**

   * Verbinden Sie sich über SSH über das Netzwerk mit der oberen Maschine und führen Sie den folgenden Code aus
   * Wenn Klipper aktualisiert wird, wird es automatisch zurückgesetzt
   * Bitte führen Sie dies nicht während des Drucks aus
   ```bash
   sed -i 's/TRSYNC_TIMEOUT = 0.025/TRSYNC_TIMEOUT = 0.05/g' ~/klipper/klippy/mcu.py && sudo systemctl restart klipper
   ```
