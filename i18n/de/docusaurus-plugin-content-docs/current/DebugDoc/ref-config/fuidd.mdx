# Pause und Abbruch des Drucks an benutzerdefinierter Position

    * Es muss `printer.cfg` hinzugefügt werden: `[include mainsail.cfg]` oder `[include fluidd.cfg]`
    * Fügen Sie die nachstehenden Makros zu `printer.cfg` hinzu

## Deutsche Übersetzung

    ```cfg
    [gcode_macro _CLIENT_VARIABLE]
    ### Einstellungen für Pause-Makros
    variable_use_custom_pos   : False ; Benutzung von benutzerdefinierten X, Y Park-Koordinaten [Wahr/Falsch]
    variable_custom_park_x    : 0.0   ; X-Position beim Pausieren des Drucks
    variable_custom_park_y    : 0.0   ; Y-Position beim Pausieren des Drucks
    variable_custom_park_dz   : 2.0   ; Z-Höhe beim Anheben während des Pausierens (in Millimetern)

    ### Einstellungen für Abbruch-Pause-Makros
    variable_park_at_cancel   : False ; Erlauben, dass der Werkzeugkopf bei der Ausführung des Abbruchdrucks zum Parkplatz bewegt wird [Wahr/Falsch]
    variable_park_at_cancel_x : None  ; X-Position nach Abbruch des Drucks
    variable_park_at_cancel_y : None  ; Y-Position nach Abbruch des Drucks

    ### Aktionen der Extruder während Pause und Abbruch
    variable_retract          : 1.0     ; Länge des Rückzugs des Extruders beim Pausieren des Drucks (mm)
    variable_cancel_retract   : 5.0     ; Länge des Rückzugs des Extruders beim Abbrechen des Drucks (mm)
    variable_speed_retract    : 35.0    ; Geschwindigkeit des Rückzugs des Extruders mm/s
    variable_unretract        : 1.0     ; Länge des Vorstoßens beim Fortsetzen des Drucks
    variable_speed_unretract  : 35.0    ; Geschwindigkeit des Vorstoßens des Extruders mm/s
    variable_speed_hop        : 15.0    ; Geschwindigkeit der Z-Achsenbewegung mm/s
    variable_speed_move       : 100.0   ; Bewegungsgeschwindigkeit mm/s

    ### Konfiguration für das Abschalten der Motorheizung
    ### Ein Tag entspricht 86400 Sekunden
    variable_idle_timeout     : 0     ; Sekunden bis zum Start des Leerlauftimeouts. 0 bedeutet, dass keine Werte gesetzt oder wiederhergestellt werden

    # !!! Achtung: Wenn Sie `use_fw_retract: True` einstellen, muss in `printer.cfg` [firmware_retraction] definiert sein !!!
    variable_use_fw_retract   : False ; Benutzung der Firmware-Rückzugssteuerung anstelle der manuellen Version [Wahr/Falsch]
    variable_runout_sensor    : ""    ; Wenn ein Sensor definiert ist, wird er verwendet, um die Ausführung von RESUME im Falle von fehlendem Filament zu unterbrechen.
    #                                Geben Sie den Konfigurationsnamen des Runout-Sensors an, z.B. "filament_switch_sensor runout". Hinweis: Verwenden Sie den gleichen wie in Ihrer printer.cfg
    # !!! Benutzerdefinierte Makros, bitte mit Vorsicht verwenden und den entsprechenden Makroabschnitt überprüfen. Diese Makros sind für einfache Operationen wie das Setzen einer Status-LED gedacht. Stellen Sie sicher, dass Ihr Makro nicht mit den grundlegenden Makrofunktionen interferiert. Es werden nur Einzeilenbefehle unterstützt, erstellen Sie ein Makro, wenn Sie mehrere Befehle benötigen.
    variable_user_pause_macro : ""    ; Alles innerhalb von "" wird nach der Klipper-Basis-Pause (PAUSE_BASE) Funktion ausgeführt
    variable_user_resume_macro: ""    ; Alles innerhalb von "" wird vor der Klipper-Basis-Fortsetzungs-Funktion (RESUME_BASE) ausgeführt
    variable_user_cancel_macro: ""    ; Alles innerhalb von "" wird vor der Klipper-Basis-Abbruch-Funktion (Abbruchdruck-Makro_BASE) ausgeführt
    gcode:
    ```

## Englisch (Originalversion)

    ```cfg
    [gcode_macro _CLIENT_VARIABLE]
    variable_use_custom_pos   : False ; use custom park coordinates for x,y [True/False]
    variable_custom_park_x    : 0.0   ; custom x position; value must be within your defined min and max of X
    variable_custom_park_y    : 0.0   ; custom y position; value must be within your defined min and max of Y
    variable_custom_park_dz   : 2.0   ; custom dz value; the value in mm to lift the nozzle when move to park position
    variable_retract          : 1.0   ; the value to retract while PAUSE
    variable_cancel_retract   : 5.0   ; the value to retract while CANCEL_PRINT
    variable_speed_retract    : 35.0  ; retract speed in mm/s
    variable_unretract        : 1.0   ; the value to unretract while RESUME
    variable_speed_unretract  : 35.0  ; unretract speed in mm/s
    variable_speed_hop        : 15.0  ; z move speed in mm/s
    variable_speed_move       : 100.0 ; move speed in mm/s
    variable_park_at_cancel   : False ; allow to move the toolhead to park while execute CANCEL_PRINT [True/False]
    variable_park_at_cancel_x : None  ; different park position during CANCEL_PRINT [None/Position as Float]; park_at_cancel must be True
    variable_park_at_cancel_y : None  ; different park position during CANCEL_PRINT [None/Position as Float]; park_at_cancel must be True
    # !!! Caution [firmware_retraction] must be defined in the printer.cfg if you set use_fw_retract: True !!!
    variable_use_fw_retract   : False ; use fw_retraction instead of the manual version [True/False]
    variable_idle_timeout     : 0     ; time in sec until idle_timeout kicks in. Value 0 means that no value will be set or restored
    variable_runout_sensor    : ""    ; If a sensor is defined, it will be used to cancel the execution of RESUME in case no filament is detected.
    #                                   Specify the config name of the runout sensor e.g "filament_switch_sensor runout". Hint use the same as in your printer.cfg
    # !!! Custom macros, please use with care and review the section of the corresponding macro.
    # These macros are for simple operations like setting a status LED. Please make sure your macro does not interfere with the basic macro functions.
    # Only  single line commands are supported, please create a macro if you need more than one command.
    variable_user_pause_macro : ""    ; Everything inside the "" will be executed after the klipper base pause (PAUSE_BASE) function
    variable_user_resume_macro: ""    ; Everything inside the "" will be executed before the klipper base resume (RESUME_BASE) function
    variable_user_cancel_macro: ""    ; Everything inside the "" will be executed before the klipper base cancel (CANCEL_PRINT_BASE) function
    gcode:
    ```
