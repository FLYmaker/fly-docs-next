Hier wird erläutert, wie man die Firmware kompiliert:
* Nach dem Verbinden über SSH geben Sie den folgenden Befehl ein und drücken Sie die Eingabetaste:

    ```bash
    cd ~/klipper && rm -rf ~/klipper/.config && rm -rf ~/klipper/out && make menuconfig
    ```

* Wobei `rm -rf ~/klipper/.config && rm -rf ~/klipper/out` die vorherigen Kompilierungsdaten und Firmware löscht. `make menuconfig` kompiliert die Firmware. Nach der Ausführung sollte die folgende Oberfläche erscheinen:

    <ImageView image={require('@site/docs/General/dfu-flash/_firmware/img/make1.png').default} size="100%" align="Center" />

* Wählen Sie **Enable extra low-level configuration options** und drücken Sie die Eingabetaste.

    <ImageView image={require('@site/docs/General/dfu-flash/_firmware/img/make2.png').default} size="100%" align="Center" />

* Gehen Sie in das Menü **Micro-controller Architecture** und wählen Sie **Raspberry Pi RP2040** und drücken Sie die Eingabetaste.

    <ImageView image={require('@site/docs/General/dfu-flash/_firmware/2040/img/2040.png').default} size="100%" align="Center" />

* Wählen Sie **Bootloader offset** und dann: **16KiB bootloader**

    <ImageView image={require('@site/docs/General/dfu-flash/_firmware/2040/img/16kb.png').default} size="100%" align="Center" />

* Wählen Sie **Communication interface** und dann: **CAN bus**

    <ImageView image={require('@site/docs/General/dfu-flash/_firmware/2040/img/can.png').default} size="100%" align="Center" />

* Wählen Sie `(4) CAN RX gpio number`, löschen Sie 4 und geben Sie 1 ein.
* Wählen Sie `(5) CAN RX gpio number`, löschen Sie 5 und geben Sie 0 ein.

    <ImageView image={require('@site/docs/General/dfu-flash/_firmware/2040/img/can1.png').default} size="100%" align="Center" />
