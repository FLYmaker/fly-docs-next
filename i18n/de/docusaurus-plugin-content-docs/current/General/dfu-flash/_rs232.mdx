    * Stellen Sie sicher, dass der Host-PC eine funktionierende Internetverbindung hat
    * Stellen Sie sicher, dass das Kabel, das den Host-PC mit dem Target-PC verbindet, Datenübertragungsfähigkeiten besitzt
    * Nach dem SSH-Verbinden zum Host-PC geben Sie `lsusb` ein und drücken Sie die Eingabetaste. Es können die folgenden drei Szenarien auftreten:
        * Es werden erkannte USB-Geräte angezeigt
        * Es wird angezeigt, dass der Befehl `ls` nicht vorhanden ist. Sie können den folgenden Befehl ausführen, um ihn zu installieren
            ```bash
            sudo apt-get install usbutils
            ```
        * Keine Fehler, keine weiteren Informationen (Systemproblem, Systemwechsel erforderlich)
    * Wenn nach Eingabe von `lsusb` keine Reaktion erfolgt, handelt es sich um ein Systemproblem, an dem wir nichts ändern können. Sie müssen ein funktionierendes System verwenden.
    * Wenn die untenstehende Abbildung die markierten Informationen zeigt, können Sie mit dem nächsten Schritt fortfahren.

        <ImageView image={require('@site/docs/General/get-id/img/df11.png').default} size="80%" align="left" /> 

    * `0843:df11` gehört zum Gerät, das Sie in diesem Vorgang verwenden. Die folgenden Hinweise können Sie ignorieren, da Systemprobleme dazu führen können, dass diese nicht vollständig angezeigt werden oder gar nicht.
    * Falls nicht, überprüfen Sie bitte selbst, ob das Datenkabel korrekt angeschlossen ist oder ob das Motherboard im DFU-Modus ist.

    :::warning Hinweise

    Gehen Sie nur zum nächsten Schritt, wenn `0843:df11` angezeigt wird.

    :::
