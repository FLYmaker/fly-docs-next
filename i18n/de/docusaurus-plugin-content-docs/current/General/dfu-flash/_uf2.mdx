    * Stellen Sie sicher, dass das obere System eine normale Internetverbindung hat
    * Stellen Sie sicher, dass das Datenkabel des unteren Systems zur oberen System-Datenübertragungsfunktion verfügt
    * Nach dem SSH-Verbindung zum oberen System eingeben `lsusb` und Enter, können drei Situationen auftreten:
        * Die erkannten USB-Geräte werden angezeigt
        * Es wird darauf hingewiesen, dass kein `ls`-Befehl vorhanden ist. Sie können diesen Befehl ausführen, um ihn zu installieren:
            ```bash
            sudo apt-get install usbutils
            ```
        * Es gibt keine Fehlermeldung und keine weiteren Informationen (Systemproblem erfordert Systemwechsel)
    * Wenn nach Eingabe von `lsusb` keine Reaktion erfolgt, ist dies ein Systemproblem, das wir nicht lösen können. Sie müssen ein normal funktionierendes System verwenden
    * Wenn die untenstehende Bildinformation wie untenstehend markiert erscheint, können Sie mit dem nächsten Schritt fortfahren
        <ImageView image={require('@site/docs/General/get-id/img/2e8a.png').default} size="80%" align="left" /> 
    * `2e8a:0003` gehört zu dem Gerät, das Sie dieses Mal verwenden. Die folgenden Hinweise müssen Sie nicht beachten, da Systemprobleme dazu führen können, dass sie unvollständig angezeigt werden oder gar nicht angezeigt werden
    * Wenn nicht, überprüfen Sie bitte selbst, ob das Datenkabel ordnungsgemäß angeschlossen ist oder ob das Hauptbord im Katapult-Modus ist (ursprünglich CANBOOT)
