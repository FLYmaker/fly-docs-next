* Bitte stellen Sie sicher, dass die Oberstation eine funktionierende Internetverbindung hat
* Bitte stellen Sie sicher, dass das Datenkabel der Unterstation zur Oberstation eine Datenübertragungsfunktion hat
* Nach der SSH-Verbindung zur Oberstation geben Sie `lsusb` ein und drücken Sie die Eingabetaste. Es können folgende drei Szenarien auftreten:
    * Es werden erkannte USB-Geräte angezeigt
    * Es wird angezeigt, dass der Befehl `ls` nicht vorhanden ist. In diesem Fall können Sie diesen Befehl ausführen, um es zu installieren:
        ```bash
        sudo apt-get install usbutils
        ```
    * Es gibt keinen Fehler, keine anderen Informationen (Systemproblem, erfordert einen Systemwechsel)
* Wenn nach Eingabe von `lsusb` überhaupt keine Reaktion erfolgt, handelt es sich um ein Systemproblem, dem wir nicht behelfen können. Sie müssen ein funktionierendes System verwenden
* Wenn die untenstehende Abbildung den markierten Bereich zeigt, können Sie mit dem nächsten Schritt fortfahren
    <ImageView image={require('@site/docs/General/get-id/img/6177.png').default} size="80%" align="left" /> 
* `1d50:6177` gehört zu dem Gerät, das Sie in diesem Fall verwenden. Die nachfolgenden Hinweise können Sie ignorieren, da Systemprobleme dazu führen können, dass diese unvollständig oder gar nicht angezeigt werden
* Falls nicht, überprüfen Sie bitte selbst, ob das Datenkabel richtig angeschlossen ist oder ob das Hauptboard sich im Katapult-Modus (ursprünglich CANBOOT) befindet
* Wenn Sie bereits die Klipper-Firmware mit Katapult geflasht haben, sehen Sie sich bitte die nachfolgende Methode zum Aktualisieren der Firmware an

    :::warning Hinweise

    Gehen Sie nur zum nächsten Schritt, wenn `1d50:6177` angezeigt wird

    :::
