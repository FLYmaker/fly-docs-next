## USB-Suchanleitung

## Vorbereitungen vor der Gerätesuche

    * Vor der Suche nach der USB-ID bitte [mit SSH verbinden](/docs/DebugDoc/BasicTutorial/index.mdx "Klicken Sie hier, um zu springen")
    * Achten Sie darauf, dass Sie sich über das Netzwerk und nicht über die serielle Schnittstelle bei SSH anmelden
    * Stellen Sie sicher, dass das Hauptboard mit dem USB-Firmware-Board korrekt verbunden ist und das Datenkabel eine Datenübertragungsfunktion hat

## Gerätesuche

    * Nachdem Sie sich erfolgreich bei der Host-Maschine angemeldet haben, können Sie `lsusb` eingeben, um das Gerät zu suchen. Es gibt folgende Szenarien:
        * Wenn `lsusb` nicht gefunden wird, können Sie den folgenden Befehl ausführen, um die entsprechende Software zu installieren:
            ```bash
            sudo apt-get install usbutils
            ```
        * Wenn `lsusb` keine Reaktion zeigt, liegt ein Systemproblem vor, das hier nicht behoben werden kann. Sie müssen das System wechseln oder ein bekannt funktionierendes System verwenden
        * Wenn Informationen wie im Bild unten angezeigt werden, achten Sie nur auf die Zeichen nach der ID:
            <ImageView image={require('@site/docs/General/get-id/img/614e.png').default} size="100%" align="left" />
    * `1d50:614e` gehört zu dem Gerät, das Sie dieses Mal verwenden wollen. Die nachfolgenden Hinweise können Sie ignorieren, da Systemprobleme dazu führen können, dass sie unvollständig oder gar nicht angezeigt werden.
    * Falls nichts gefunden wird, überprüfen Sie bitte, ob das Datenkabel richtig angeschlossen ist und ob die richtige Firmware installiert ist.
    
:::warning Hinweise
Suchen Sie die USB-ID nur, wenn `1d50:614e` vorhanden ist.
:::

## USB-ID suchen

    ```bash
    ls /dev/serial/by-id/*
    ```

    * Geben Sie den untenstehenden Befehl ein und drücken Sie Enter. Bei korrekter Funktion wird eine blaue ID angezeigt. (Beispiel unten)
    <ImageView image={require('@site/docs/General/get-id/img/usb-id.png').default} size="80%" align="left" />
    * Wenn nach Eingabe des Befehls keine ID erscheint, aber `lsusb` ein Gerät anzeigt, wechseln Sie die Host-Maschine oder das Betriebssystem der Host-Maschine.


:::warning Hinweise
Falls mehrere IDs erscheinen, wählen Sie die entsprechende ID entsprechend dem **MCU**-Modell Ihres Hauptboards aus.
:::

### USB-ID in die Konfiguration einfügen

    * Öffnen Sie die Drucker-Weboberfläche, suchen Sie im linken Seitenbereich nach den Konfigurationsoptionen: fluidd ist: `{…}` ; mainsail ist: `Maschine`, klicken Sie darauf, um `printer.cfg` zu finden

        <ImageView image={require('@site/docs/General/get-id/img/fluidd-cfg.png').default} size="50%" align="left" />

    * Kopieren Sie die blaue ID und fügen Sie sie in `printer.cfg` in den `[mcu]`-Knoten unter `serial: ` ein. Speichern und neustarten Sie, um das Hauptboard zu verbinden. Wenn Klipper `ADC out of range` anzeigt, ist dies normal. Schließen Sie das Heizbett und den Temperatursensor an, konfigurieren Sie die entsprechenden Pins für Extruder und Heizbett, speichern und neustarten Sie erneut.

            <ImageView image={require('@site/docs/General/get-id/img/cfg-usb-id.png').default} size="50%" align="left" />

:::danger
Hinweis: Alle in diesem Dokument gezeigten IDs sind Beispiele. Jede Hauptplatine hat eine eigene, individuelle ID. Bitte passen Sie die ID entsprechend Ihrer tatsächlichen ID an.
:::
