# Hinzufügen eines drahtlosen Buttons

    :::info Wichtiger Hinweis
    * Der drahtlose Button benötigt die Installation einer Batterie, der Batterietyp ist `CR927`
    :::

    * Kehren Sie zur Startseite zurück und klicken Sie auf das `+` im roten Rahmen im Bild
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-airclick/img/button.png').default} size="80%" align="Center" />
    * Wenn die folgende Oberfläche erscheint, drücken Sie normal den drahtlosen Button, nachdem die Batterie installiert ist
    * Im roten Rahmen wird automatisch die ID des drahtlosen Buttons erzeugt, diese ID ist einzigartig
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-airclick/img/button-1.png').default} size="80%" align="Center" />
    * Geben Sie im roten Rahmen den Namen ein, den Sie dem Button geben möchten
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-airclick/img/button-2.png').default} size="80%" align="Center" />
    * Klicken Sie auf den ersten roten Rahmen, es erscheint ein zweiter blauer Rahmen
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-airclick/img/button-3.png').default} size="80%" align="Center" />



    :::danger Wichtige Information

    * Benutzerdefinierte GCODE und API-Anforderungen werden über die drahtlose Verbindung gesteuert
    * Das Auslösen von GPIO erfolgt über eine Kabelverbindung

    :::

## Benutzerdefinierter GCODE
    * Sie können benutzerdefinierte `GCODE` eingeben, indem Sie den Code oder den Makronamen in den roten Rahmen eingeben, AirClick sendet diesen automatisch an Klipper zur Ausführung
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-airclick/img/gcode.png').default} size="80%" align="Center" />

## API-Anforderung
    * Sie können benutzerdefinierte Makros, Notstopps, Neustarts von Klipper, Firmware-Neustarts und das Herunterfahren des Systems umsetzen
        <ImageView image={require('@site/docs/ProductDoc/ExtensionBoard/fly-airclick/img/api.png').default} size="80%" align="Center" />

## Auslösen von GPIO
    * Das Auslösen von GPIO erfordert eine Verbindung von STM32F072 mit Klipper
    * Für Firmware-Brennprozesse und Verbindungen, bitte überprüfen Sie dieses Dokument <Button variant="contained" disableElevation href="/docs/ProductDoc/ExtensionBoard/fly-airclick/offline-operation-support">Firmware-Brennen und Verbindung</Button>
    * Die nutzbaren IOs sind `PB0` bis `PB9`, insgesamt zehn IOs, nach dem normalen Brennen der Firmware und der Verbindung zu Klipper benötigen Sie nur eine normale Konfiguration, hier ist eine Beispielkonfiguration

        ```cfg
        [mcu AirClick]
        serial: /dev/serial/by-id/usb-Klipper_stm32f072xb_AirClick-if00
        ### Sie müssen diese Zeile nach der USB-ID-Suche ändern

        [gcode_button _test]
        pin: ^!AirClick:PB0
        press_gcode:
            M118 PB0

        [gcode_button _test1]
        ```
