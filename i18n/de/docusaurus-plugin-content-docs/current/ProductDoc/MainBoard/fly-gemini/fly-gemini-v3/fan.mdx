# Einbauter 5V-Lüfteranschluss

## Gemini hat eine 5V-Steuerungsschnittstelle für die Kühlung des Hauptcontrollers, der Anschluss ist wie im folgenden Bild dargestellt:

    <ImageView image={require('@site/docs/ProductDoc/MainBoard/fly-gemini/fly-gemini-v3/img/corefan.png').default} size="80%" align="center" />

## Referenz-Konfiguration

    ```cfg
    [mcu host]                      # FlyOS hat standardmäßig einen Linux-MCU für Sie erstellt, definieren Sie ihn einfach zur Verwendung           
    serial: /tmp/klipper_host_mcu 

    [temperature_sensor FLY-π]      # Temperatur des Host-Systems
    sensor_type: temperature_host


    [temperature_fan core_fan]      # Lüfter für die Kühlung des Host-Systems
    pin: host:gpiochip1/gpio103     # Pin für den Lüfter des Host-Systems
    max_power: 1.0
    sensor_type: temperature_host   # Setzen Sie auf die Hauptkontrollertemperatur des Host-Systems
    control:watermark               # Kontrollmethode
    target_temp: 48                 # Starttemperatur des Lüfters des Host-Systems
    min_temp: 0                     # Mindesttemperatur, unterhalb dieser Temperatur wird ein Fehler gemeldet
    max_temp: 90                    # Maximaltemperatur, oberhalb dieser Temperatur wird ein Fehler gemeldet
    off_below: 0.10
    kick_start_time: 0.50
    max_speed: 0.8                  # Maximale Drehzahl, 80% der Maximalleistung
    min_speed: 0.3                  # Minimale Drehzahl, 30% der Maximalleistung
    ```
