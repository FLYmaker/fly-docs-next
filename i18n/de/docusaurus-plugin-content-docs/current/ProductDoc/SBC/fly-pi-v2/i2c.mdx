# I2C-Nutzung

    * Verwenden Sie die I2C-Schnittstelle des Host-PCs über 40PIN
    * Diese Anleitung verwendet den I2C-Sensor FLY-PI V2LM75 und I2C2 als Beispiel!!!
    * Beachten Sie, dass I2C1 nicht verwendet werden kann, wenn es von anderen Geräten belegt ist!!!
    * Stellen Sie sicher, dass die Verkabelung stabil und fest ist!!!

## Öffnen von I2C über FLY Tools

    * Finden Sie Overlays und wählen Sie i2c2 und i2c0 aus
    * Beachten Sie, dass I2C1 nicht verwendet werden kann, wenn es von anderen Geräten belegt ist!!!
    * Speichern Sie und starten Sie neu

## Verkabelung und Suche

    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-pi-v2/img/i2c2.png').default} size="80%" align="center" />

    * Verbinden Sie sich über SSH mit dem Host-PC und geben Sie den untenstehenden Befehl ein, um den folgenden Inhalt auszugeben
    * Wenn `I2C0` verwendet wird, ersetzen Sie die `2` im Befehl unten durch `0`

    ```cfg
    sudo i2cdetect -y -r 2
    ```

    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-pi-v2/img/i2c1.png').default} size="80%" align="center" />

    * Beachten Sie, dass bei normalem Betrieb sofort der obenstehende Inhalt mit einer Adresse angezeigt wird
    * Beachten Sie, dass die Adresse zufällig ist
    * Beachten Sie, dass bei falscher Verkabelung, vertauschten Leitungen, nicht geöffneter Konfiguration oder beschädigtem Modul die Ausgabe nicht normal ist
    * Die Adresse muss von Hexadezimal in Dezimal umgerechnet werden, um sie in die Konfiguration hinter `i2c_address:` einzutragen
    * Sie können einen Online-Basiskonverter im Browser verwenden, um die Umrechnung durchzuführen

## Konfiguration

    ```cfg
    [mcu host]
    serial: /tmp/klipper_host_mcu

    [temperature_sensor my_sensor]
    sensor_type: LM75
    i2c_address:72
    # #   Standardwert ist 72 (0x48). Normaler Bereich ist 72-79 (0x48-0x4F) und die 3
    # #   niedrigen Bits der Adresse werden über Pins auf dem Chip konfiguriert
    # #   (normalerweise über Jumper oder fest verdrahtet).
    i2c_mcu:host
    i2c_bus:i2c.2
    # i2c_software_scl

    ```
