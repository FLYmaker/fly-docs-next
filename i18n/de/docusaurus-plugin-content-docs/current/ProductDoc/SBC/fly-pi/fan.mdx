# Onboard 5V Fan Connector

## Der Pi hat ein 5V steuerbares Interface für die Kühlung des Hauptcontrollers, die Verkabelungsmethode wie im Folgenden gezeigt:

    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-pi/img/corefan.png').default} size="80%" align="center" />

## Referenzkonfiguration

    ```cfg
    [mcu host]                      # FlyOS hat standardmäßig eine Linux-MCU für Sie generiert, die nach Definition verwendet werden kann
    serial: /tmp/klipper_host_mcu 

    [temperature_sensor FLY-π]      # Temperatur des Hostcomputers
    sensor_type: temperature_host


    [temperature_fan core_fan]      # Lüfter für die Kühlung des Hostcomputers
    pin: host:gpiochip1/gpio103     # Pin für den Hostcomputer-Lüfter
    max_power: 1.0
    sensor_type: temperature_host   # Setze als Temperatur des Hauptcontrollers
    control:watermark               # Steuerungsart
    target_temp: 48                 # Temperatur, bei der der Hostcomputer-Lüfter startet
    min_temp: 0                     # Mindesttemperatur, unterhalb derer ein Fehler angezeigt wird
    max_temp: 90                    # Höchsttemperatur, oberhalb derer ein Fehler angezeigt wird
    off_below: 0.10
    kick_start_time: 0.50
    max_speed: 0.8                  # Maximale Drehzahl, 80% der vollen Leistung
    min_speed: 0.3                  # Minimale Drehzahl, 30% der vollen Leistung
    ```
