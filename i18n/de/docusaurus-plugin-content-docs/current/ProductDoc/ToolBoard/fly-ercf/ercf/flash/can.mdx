# CAN-Firmware-Kompilierung

## Kompilierung beginnen

    <SSH />

    <MakeOperate/>

    <CAN />

    * Wählen Sie GPIO-Pins zur Festlegung beim Mikrocontroller-Start, Eingabe: gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/can.png').default} size="100%" align="center" />
    * Drücken Sie die `Q`-Taste, **Konfiguration speichern** erscheint, dann drücken Sie die `Y`-Taste
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Die Konfiguration sollte jetzt gespeichert und Sie zurück zum Kommandozeilen-Interface geführt haben
    * Geben Sie `make -j4` ein, um mit der Kompilierung zu beginnen, dies dauert etwas länger
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Wenn schließlich der folgende Inhalt ausgegeben wird, war die Kompilierung erfolgreich

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Firmware programmieren

    * Verbinden Sie die Toolboard mit UTOC oder einem Hauptboard, das mit der Bridge-Firmware programmiert wurde, verwenden Sie dafür das XT30-Datenkabel. Unten finden Sie ein Referenzverdrahtungsdiagramm
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Stellen Sie sicher, dass CAN H und CAN L unter Stromausfall mit einem Multimeter gemessen werden und ein Widerstandswert von etwa `60Ω` aufweisen. Ist der Wert `120Ω`, müssen Sie den `120Ω`-Jumper anschließen, ist er etwa `40Ω`, muss der Jumper entfernt werden
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercf/flash/img/120.png').default} size="50%" align="center" />
