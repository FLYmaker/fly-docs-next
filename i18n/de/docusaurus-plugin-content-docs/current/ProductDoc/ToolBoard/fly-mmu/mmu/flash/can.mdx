    * Wählen Sie GPIO-Pins zum Setzen beim Mikrocontroller-Start, Eingabe: !PA15 Beachten Sie bitte, dass ! großgeschrieben ist
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/can.png').default} size="100%" align="center" />
    * Drücken Sie die `Q`-Taste, wenn **Save configuration** erscheint, dann die `Y`-Taste
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Die Konfiguration sollte jetzt gespeichert und die Konsole verlassen sein
    * Geben Sie `make -j4` ein, um die Compilation zu starten, es kann eine Weile dauern
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Bei der folgenden Ausgabe wurde die Compilation erfolgreich abgeschlossen:

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Firmware brennen

    * Verbinden Sie die Werkzeugplatte mit dem UTOC oder einer Hauptplatine mit gebrückter Firmware vor dem Brennen der Firmware mit einem XT30-Datenkabel, unten ist ein Referenzanschlussdiagramm
    * Achten Sie darauf, dass der DIP-Schalter korrekt eingestellt ist
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/can_mode.png').default} size="90%" align="center" />
    * Verbindung zum UTOC
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Stellen Sie sicher, dass der Widerstand zwischen CAN H und CAN L bei ausgeschalteter Stromversorgung mit einem Multimeter gemessen etwa `60Ω` beträgt; wenn es `120Ω` ist, muss der `120Ω`-Jumper angeschlossen werden, bei etwa `40Ω` muss der Jumper entfernt werden
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-mmu/mmu/flash/img/120.png').default} size="50%" align="center" />
