# CAN-Firmware-Kompilierung

## Kompilierung starten

    <SSH />

    <MakeOperate/>

    <CAN />

    * Wähle **GPIO pins to set at micro-controller startup** aus und gib ein: !gpio5 Beachte, dass ! großgeschrieben ist
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/can.png').default} size="100%" align="center" />
    * Drücke die `Q`-Taste, es erscheint **Save configuration**, dann drücke die `Y`-Taste
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Jetzt sollte die Konfiguration gespeichert und die Konsole verlassen sein
    * Gib `make -j4` ein, um mit der Kompilierung zu beginnen, dies kann etwas dauern
    <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Wenn die Kompilierung erfolgreich war, wird folgendes ausgegeben:

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Firmware flashen

    * Vor dem Flashen der Firmware, verbinde das Toolboard mit einem UTTO oder einem Hauptboard, das die Bridge-Firmware geladen hat, mit einem XT30-Datenkabel. Hier ist ein Referenzbild des Anschlusses
    * Achte darauf, dass die DIP-Schalter korrekt eingestellt sind
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Stelle sicher, dass der Widerstand zwischen CAN H und CAN L bei ausgeschaltetem Strom mit einem Multimeter gemessen wird und etwa `60Ω` beträgt. Wenn der Widerstand `120Ω` ist, muss der `120Ω`-Schalter aktiviert werden, wenn er etwa `40Ω` beträgt, muss der Schalter deaktiviert werden
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/120.png').default} size="100%" align="center" />
