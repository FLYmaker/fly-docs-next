# RS232-Firmware-Programmierung

## Programmierung starten

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Wählen Sie GPIO-Pins zum Setzen beim Mikrocontroller-Start, geben Sie ein: !gpio5 Achten Sie darauf, dass ! großgeschrieben ist
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/rs232.png').default} size="100%" align="center" />
    * Drücken Sie die `Q` Taste, es erscheint **Save configuration**, dann drücken Sie die `Y` Taste
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Jetzt sollte die Konfiguration gespeichert und Sie sollten wieder auf der Kommandozeile sein
    * Geben Sie `make -j4` ein, um mit der Programmierung zu beginnen, das kann etwas Zeit in Anspruch nehmen
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Wenn schließlich der folgende Inhalt ausgegeben wird, war die Programmierung erfolgreich

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```


## Programmiermodus betreten

    :::tip

    Links befindet sich die `RESET-Taste`, rechts die `BOOT-Taste`

    :::

    * Es gibt zwei Methoden, um in den DFU-Modus zu gelangen
    * Erste Methode: Schalten Sie die Hauptplatine vollständig aus, drücken Sie die BOOT-Taste und verbinden Sie dann das Datenkabel mit der Hauptplatine und dem Host-Computer
    * Zweite Methode: Schließen Sie das Datenkabel an die Hauptplatine und den Host-Computer an, drücken Sie die BOOT-Taste und dann die RST-Taste, lassen Sie die RST-Taste los und dann die BOOT-Taste
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-v3/flash/img/boot.png').default} size="100%" align="center" />
