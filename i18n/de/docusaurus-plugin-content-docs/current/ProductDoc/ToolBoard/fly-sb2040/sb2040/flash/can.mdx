# CAN-Firmware-Kompilierung

## Kompilierung starten

    <SSH />

    <MakeOperate/>

    <CAN />

    * Wählen Sie GPIO-Pins aus, die beim Start des Mikrocontrollers gesetzt werden sollen, und geben Sie ein: gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/can.png').default} size="100%" align="center" />
    * Drücken Sie die `Q`-Taste, es erscheint **Konfiguration speichern**, dann drücken Sie die `Y`-Taste
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Die Konfiguration sollte jetzt gespeichert sein und Sie sollten zur Kommandozeile zurückkehren
    * Geben Sie `make -j4` ein, um mit der Kompilierung zu beginnen, die etwas Zeit in Anspruch nehmen kann
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Am Ende wird der folgende Inhalt ausgegeben, wenn die Kompilierung erfolgreich war

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Firmware-Programmierung

    * Verbinden Sie die Toolboard mit einem UTOC oder einem Mainboard mit der Brücke-Firmware, bevor Sie die Firmware programmieren, unten ist das Referenzanschlussdiagramm
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.png').default} size="80%" align="left" />
    * Stellen Sie sicher, dass die Widerstandswerte von CAN H und CAN L im ausgeschalteten Zustand mit einem Multimeter gemessen etwa `60Ω` betragen, wenn es `120Ω` ist, müssen Sie den `120Ω`-Jumper anschließen, wenn es etwa `40Ω` ist, müssen Sie den Jumper entfernen
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/120.png').default} size="50%" align="center" />
