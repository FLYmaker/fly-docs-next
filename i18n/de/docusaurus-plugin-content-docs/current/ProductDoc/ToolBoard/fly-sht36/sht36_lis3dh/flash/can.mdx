## Kompilierung starten

    <SSH />

    <CAN />

    * Wählen Sie GPIO-Pins für den Mikrocontroller-Start, geben Sie ein: !gpio20 Achten Sie darauf, dass ! großgeschrieben ist
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.png').default} size="100%" align="center" />
    * Drücken Sie die `Q`-Taste, es erscheint **Save configuration**, dann drücken Sie die `Y`-Taste
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Die Konfiguration sollte jetzt gespeichert und die Konsole verlassen sein
    * Geben Sie `make -j4` ein, um die Kompilierung zu starten, es dauert etwas
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Wenn am Ende die folgenden Zeilen ausgegeben werden, war die Kompilierung erfolgreich

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Firmware flashen

    * Vor dem Flashen der Firmware die Toolboard mit einem XT30-Kabel an den UTOC oder das Hauptboard mit Bridge-Firmware anschließen. Unten ist das Anschlussschema
    * Achten Sie darauf, dass der DIP-Schalter auf die richtige Position gestellt ist
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Stellen Sie sicher, dass der Widerstand zwischen CAN H und CAN L im Stromlosen Zustand mit einem Multimeter gemessen wird und bei etwa `60Ω` liegt. Wenn er `120Ω` beträgt, muss ein `120Ω`-Jumper angeschlossen werden; wenn er etwa `40Ω` beträgt, muss der Jumper entfernt werden
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.png').default} size="50%" align="center" />
