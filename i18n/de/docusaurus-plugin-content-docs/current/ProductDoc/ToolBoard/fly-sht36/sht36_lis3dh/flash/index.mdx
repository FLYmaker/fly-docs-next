# Firmware-Flash-Anleitung

    * SHT36 LIS3DH können nur über CAN-Verbindung mit dem Host-Computer verbunden werden
    * CAN benötigt eine `XT30(2+2)-F`-Verbindung
    * Unten ist das Aussehen des `XT30(2+2)-F`-Datenkabels
        <ImageView image={require('@site/docs/General/get-id/img/xt30.png').default} size="80%" align="left" />

:::danger Wichtig
* Die Hauptplatinenfirmware muss über Katapult gebrannt werden
* Wenn feststeht, dass die Hauptplatine bereits Katapult installiert hat, ist das Brennen der Katapult-Firmware am Ende nicht nötig; das ist nur eine Vorsichtsmaßnahme für den Fall, dass das Brennen der Katapult-Firmware auf der Hauptplatine fehlschlägt
* Man kann schnell zweimal zurücksetzen und schauen, ob eine LED auf der Hauptplatine aufleuchtet; wenn ja, bedeutet dies, dass die Hauptplatine in den Katapult-Modus gewechselt hat
:::

## Wie man feststellt, welche Firmware benötigt wird

    * SHT36 LIS3DH können auf folgende Weise mit dem Host-Computer verbunden werden
    * <Button variant="contained" disableElevation href="./can">CAN-Firmware brennen</Button>
        * Vorteile: Wenn die `Katapult`-Firmware installiert ist, kann die Firmware ohne Berührung der Hauptplatine aktualisiert werden und unterstützt eine Bus-Verbindung
        * Nachteile: Kann bei Verwendung zusammen mit `KlipperScreen` zu Fehlern wie Homing-Timeout führen.
        * Hinweis: Wenn CAN benötigt wird, stellen Sie sicher, dass der Host-Computer CAN unterstützt und UTOC oder eine Hauptplatine mit Bridge-Firmware erforderlich ist
