# Firmware-Brennhinweise

    * Das SHT36 PRO kann über eine CAN-Verbindung mit dem Hostcomputer verbunden werden
    * CAN benötigt eine Verbindung über `XT30(2+2)-F`
    * Unten ist das Datenkabel `XT30(2+2)-F` dargestellt
        <ImageView image={require('@site/docs/General/get-id/img/xt30.png').default} size="80%" align="left" />

:::danger Wichtig
* Der Firmware-Brennprozess für das Mainboard erfordert das Brennen von Klipper über Katapult
* Wenn sicher ist, dass das Mainboard bereits Katapult gebrannt hat, ist das Brennen der Katapult-Firmware am Ende nicht erforderlich; es ist nur eine Vorsichtsmaßnahme für den Fall, dass das Brennen der Katapult-Firmware auf dem Mainboard nicht erfolgreich war
* Man kann zweimal schnell hintereinander auf Reset drücken und überprüfen, ob eine LED auf dem Mainboard aufblitzt; wenn ja, bedeutet dies, dass das Mainboard in die Katapult-Firmware eingetreten ist
:::

## CAN-Firmware-Kompilierung

    * Das SHT36 PRO kann nur über eine CAN-Verbindung mit dem Hostcomputer verbunden werden
    * <Button variant="contained" disableElevation href="./can">CAN-Firmware brennen</Button>
        * Vorteil: Wenn die `Katapult`-Firmware gebrannt wurde, kann die Firmware ohne Berührung des Mainboards aktualisiert werden, und es unterstützt Bus-Verbindungen
        * Nachteil: Kann bei der gemeinsamen Nutzung mit `KlipperScreen` zu Fehlern wie Zeitüberschreitung bei der Homing führen.
        * Hinweis: Wenn CAN benötigt wird, stellen Sie sicher, dass der Hostcomputer CAN unterstützt und verwenden Sie UTOC oder ein Mainboard mit gebrannter Brücken-Firmware
