# Kompilierung der CAN-Firmware für Version 103

## Beginnen Sie mit der Kompilierung

    <SSH />

    <MakeOperate/>

    <CAN />

    * Wählen Sie **GPIO-Pins zum Setzen beim Start des Mikrocontrollers** aus und geben Sie ein: !PC13 Achten Sie darauf, dass es Großbuchstaben sind
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/103.png').default} size="100%" align="center" />
    * Drücken Sie die `Q`-Taste, es erscheint **Save configuration**, dann drücken Sie die `Y`-Taste
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * Die Konfiguration sollte nun gespeichert und Sie sollten wieder zur Befehlszeile zurückgekehrt sein
    * Geben Sie `make -j4` ein, um mit der Kompilierung zu beginnen, die etwas Zeit in Anspruch nimmt
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Wenn am Ende der folgende Inhalt ausgegeben wird, war die Kompilierung erfolgreich

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Firmware flashen

    * Verbinden Sie die Werkzeugplatine mit UTOC oder einem Hauptboard, das die Bridge-Firmware hat, vor dem Flashen der Firmware, hier ist ein Referenzanschlussdiagramm
    * Achten Sie darauf, dass der DIP-Schalter auf die entsprechende Position gestellt ist
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.png').default} size="80%" align="left" />
    * Stellen Sie sicher, dass CAN H und CAN L im ausgeschalteten Zustand mit einem Multimeter gemessen einen Widerstand von ca. `60Ω` haben. Wenn es `120Ω` ist, muss ein `120Ω`-Brückenstecker angeschlossen werden; wenn es um `40Ω` ist, muss der Brückenstecker entfernt werden
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.png').default} size="50%" align="center" />
