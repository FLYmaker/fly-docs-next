# Eddy Usage

    :::info

    * Please note that bed scanning is recommended to use a separate Z limit, do not use bed scanning as Z limit
    * When installing Eddy, the bottom should be at least 1-2 mm above the nozzle
    * When calibrating Eddy frequency, keep the coil PCB at least 5mm away from the heated bed
    * Please ensure the entire bed is at room temperature for testing, and do not turn on the bed during testing

    :::

## Reference Configuration and Usage Instructions

### Modifying the default frequency of ldc16121612
    This method is only for sht36-v3/SB2040-v3 or SB2040-Pro-v3
    You can use the Mobaxterm tool to execute commands

    Download link: https://mobaxterm.mobatek.net/download.html
    
    [For detailed usage, please refer](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Execute the first command and press Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Execute the second command and press Enter
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    If the replacement is successful, a line containing `LDC1612_FREQ = 40000000` will be returned, indicating success. If there is no return, it means the replacement was not successful.
    Alternatively, use this command to check for success
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```
   <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/ldc161240mhz.webp').default} size="30%" align="left" />

   Friendly reminder: After modification, please restart the Klipper service with this command and recalibrate the Eddy frequency
   ```cfg
   systemctl restart klipper
   ```
   Eddy frequency calibration command
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```

    ### Bed Scanning Configuration

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 0.8
        i2c_address: 43
        i2c_mcu: SB2040
        i2c_bus: i2c1b
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Please note that there is currently no external 1612 module available, so this configuration cannot be used at the moment
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 0.8
        i2c_address: 42
        i2c_mcu: SB2040
        i2c_bus: i2c1b
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Temperature Compensation Configuration

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SB2040:gpio28
        ```

### bed_mesh Configuration

    * In [bed_mesh], it is recommended to set the `horizontal_move_z` parameter to 2, allowing Eddy to be as close to the platform as possible during scanning

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
