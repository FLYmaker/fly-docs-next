    * Select GPIO pins to set at micro-controller startup, input: gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/can.png').default} size="100%" align="center" />
    * Press the `Q` key, then when **Save configuration** appears, press `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * The configuration should now be saved and you should be back to the command line interface
    * Enter `make -j4` to start compiling; this might take some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * If the following content is output at the end, the compilation was successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flash the Firmware

    * Before flashing the firmware, connect the tool board to UTOC or the main board with a bridge firmware using an XT30 data cable. Here is the reference wiring diagram:
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.png').default} size="80%" align="left" />
    * Ensure that the resistance between CAN H and CAN L measures around `60立` with a multimeter when powered off. If it reads `120立`, you need to connect the `120立` jumper; if it's around `40立`, remove the jumper.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/120.png').default} size="50%" align="center" />
