# Compiling CAN Firmware for Version 072

## Start Compiling

    <SSH />

    <CAN />

    * Select **GPIO pins to set at micro-controller startup**, input: !PC13 (please note it is uppercase)
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/072.png').default} size="100%" align="center" />
    * Press the `Q` key, when **Save configuration** appears, press `Y` key
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * The configuration should now be saved, and you should be back to the command line interface
    * Enter `make -j4` to start compiling, which might take some time
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * If the following content is output at the end, the compilation was successful

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashing the Firmware

    * Before flashing the firmware, use the XT30 data cable to connect the tool board to UTOC or a motherboard with bridge firmware flashed. Below is a reference wiring diagram
    * Please check if the DIP switches are set to the correct positions
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.png').default} size="80%" align="left" />
    * Ensure that in a powered-off state, use a multimeter to measure the resistance between CAN H and CAN L to be around `60立`. If it's `120立`, you need to connect the `120立` jumper. If it's around `40立`, you need to remove the jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.png').default} size="50%" align="center" />
