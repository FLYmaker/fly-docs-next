# Port de ventilateur 5V embarqué

## La carte C8 est équipée d'une interface contrôlable en 5V pour le refroidissement du contrôleur principal. Le branchement est illustré ci-dessous

    <ImageView image={require('@site/docs/ProductDoc/MainBoard/fly-c/fly-c8/img/corefan.png').default} size="80%" align="center" />

## Configuration de référence

    ```cfg
    [mcu host]                      # FlyOS a déjà généré pour vous un MCU Linux, définissez-le simplement pour l'utiliser           
    serial: /tmp/klipper_host_mcu 

    [temperature_sensor FLY-π]      # Température de l'hôte
    sensor_type: temperature_host


    [temperature_fan core_fan]      # Ventilateur de refroidissement de l'hôte
    pin: host:gpiochip1/gpio103     # Broche du ventilateur de l'hôte
    max_power: 1.0
    sensor_type: temperature_host   # Défini comme température du contrôleur principal de l'hôte
    control:watermark               # Mode de contrôle
    target_temp: 48                 # Température de déclenchement du ventilateur de refroidissement de l'hôte
    min_temp: 0                     # Température minimale, en dessous de laquelle une erreur sera signalée
    max_temp: 90                    # Température maximale, au-delà de laquelle une erreur sera signalée
    off_below: 0.10
    kick_start_time: 0.50
    max_speed: 0.8                  # Vitesse maximale, 80% de la puissance maximale
    min_speed: 0.3                  # Vitesse minimale, 30% de la puissance maximale
    ```
