# Étendre la commande des LED via des plugins

Vous devez vous connecter en SSH à votre machine hôte et suivre les instructions ci-dessous pour l'installation et la configuration

```
https://github.com/julianschill/klipper-led_effect
```

## Installation

Veuillez utiliser **MobaXterm_Personal** ou d'autres **outils SSH** pour vous connecter via **WIFI** à votre machine hôte, et assurez-vous des points suivants :

1. **Assurez-vous que le firmware Klipper est installé sur votre machine hôte**
2. **Assurez-vous que vous vous connectez en tant que l'utilisateur ayant installé Klipper**
3. **Assurez-vous que votre méthode de saisie est en anglais**
4. **Assurez-vous que votre machine hôte peut correctement détecter l'appareil**
5. **Assurez-vous que tous ces points sont respectés, sinon vous ne pourrez pas continuer**

Ce module peut être installé sur une installation Klipper existante en utilisant un script d'installation.

```
cd ~
git clone https://github.com/julianschill/klipper-led_effect.git
cd klipper-led_effect
./install-led_effect.sh
```

## Configuration de référence

```
[neopixel TP_led]
pin: PB7
chain_count: 25
# Nombre de LEDs
# Nombre de LED
color_order: GRB
initial_RED: 0.4    
initial_GREEN: 0.8
initial_BLUE: 1
initial_WHITE: 0.0
#66CCFF 

[led_effect sb_nozzle_cooling]
autostart:              false
frame_rate:             24
leds:
    neopixel:TP_led (9,10)
layers:
        breathing  3 1 top (0.0, 0.0, 1.0, 0.1)

[led_effect rainbow]
leds:
    neopixel:TP_led
autostart:                          true
frame_rate:                         24
layers:
    gradient  0.3  1 add (0.3, 0.0, 0.0),(0.0, 0.3, 0.0),(0.0, 0.0, 0.3)

```
