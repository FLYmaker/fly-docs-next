# Connecteur de ventilateur embarqué 5V

## Pi-V2 dispose d'une interface contrôlable de 5V pour la dissipation thermique du contrôleur principal, le branchement est illustré ci-dessous
    <ImageView image={require('@site/docs/ProductDoc/SBC/fly-pi-v2/img/fan.png').default} size="80%" align="left" />


## Configuration de référence

    ```cfg
    [mcu host]                      # FlyOS a généré un MCU Linux pour vous par défaut, il suffit de le définir pour l'utiliser           
    serial: /tmp/klipper_host_mcu 

    [temperature_sensor FLY-π]      # Température de l'hôte
    sensor_type: temperature_host


    [temperature_fan core_fan]      # Ventilateur de refroidissement de l'hôte
    pin: host:gpiochip1/gpio103     # Broche du ventilateur de l'hôte
    max_power: 1.0
    sensor_type: temperature_host   # Défini comme température du contrôleur principal de l'hôte
    control:watermark               # Mode de contrôle
    target_temp: 48                 # Température de démarrage du ventilateur de refroidissement de l'hôte
    min_temp: 0                     # Température minimale, en dessous de laquelle une erreur sera signalée
    max_temp: 90                    # Température maximale, au-dessus de laquelle une erreur sera signalée
    off_below: 0.10
    kick_start_time: 0.50
    max_speed: 0.8                  # Vitesse maximale, à 80% de la puissance maximale
    min_speed: 0.3                  # Vitesse minimale, à 30% de la puissance maximale
    ```
