# Compilation du firmware CAN

## Début de la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Choisir les GPIO pins à définir au démarrage du micro-contrôleur, entrer : gpio17
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyer sur la touche `Q`, **Save configuration** apparaît, appuyer ensuite sur la touche `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et la fenêtre de commande devrait être quittée
    * Entrer `make -j4` pour commencer la compilation, cela prendra un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si le message suivant apparaît, la compilation est réussie

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outils à l'UTOC ou à la carte mère avec le firmware de pont à l'aide d'un câble de données XT30, ci-dessous un schéma de connexion de référence
    * Assurez-vous que les cavaliers sont à la position correcte
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsque l'alimentation est coupée. Si elle est de `120Ω`, connectez le shunt `120Ω`. Si elle est d'environ `40Ω`, retirez le shunt.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-ercf/ercfv2/flash/img/120.webp').default} size="100%" align="center" />
