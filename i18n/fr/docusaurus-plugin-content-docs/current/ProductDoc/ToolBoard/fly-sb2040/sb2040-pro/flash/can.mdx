# Compilation du firmware CAN

## Début de la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez GPIO pins to set at micro-controller startup, entrez : gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/can.png').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis sur `Y` lorsque **Save configuration** apparaît
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez être retourné à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Si la compilation est réussie, vous verrez le message suivant :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez le panneau d'outils à UTOC ou à la carte mère avec un câble de données XT30, voici un schéma de câblage de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.png').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsque l'appareil est éteint. Si elle est de `120Ω`, connectez un shunt de `120Ω`. Si elle est d'environ `40Ω`, retirez le shunt.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro/flash/img/120.png').default} size="50%" align="center" />
