## Début de la compilation

    <SSH />

    * Sélectionnez les broches GPIO à définir au démarrage du microcontrôleur, entrez : gpio24
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis sur `Y` pour **Save configuration**
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez revenir à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la compilation réussit, vous verrez la sortie suivante :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outils à l'UTOC ou à une carte mère avec un firmware de pont via un câble de données XT30, ci-dessous se trouve un schéma de câblage de référence
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/sb2040_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` à l'aide d'un multimètre lorsque l'appareil est éteint. Si elle est de `120Ω`, vous devez ajouter un cavalier de `120Ω`. Si elle est d'environ `40Ω`, vous devez retirer le cavalier.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040/flash/img/120.webp').default} size="50%" align="center" />
