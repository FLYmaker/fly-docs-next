# Compilation du firmware CAN

## Début de la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez GPIO pins to set at micro-controller startup, entrez : !gpio20 Veuillez noter que ! est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.png').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis sur **Save configuration**, et ensuite sur la touche `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez être revenu à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * Si la compilation est réussie, vous verrez la sortie suivante :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outils à l'UTOC ou à la carte mère avec un câble de données XT30, ci-dessous un schéma de connexion de référence
    * Veuillez vérifier si les cavaliers sont bien positionnés
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L est d'environ `60Ω` lorsque l'alimentation est coupée. Si elle est de `120Ω`, vous devez connecter le cavalier de `120Ω`. Si elle est d'environ `40Ω`, vous devez retirer le cavalier.
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.png').default} size="50%" align="center" />
