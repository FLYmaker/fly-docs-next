# Compilation du firmware CAN

## Début de la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez les broches GPIO à définir au démarrage du microcontrôleur, entrez : !gpio13 Veuillez noter que ! est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis sur **Save configuration** apparaît, appuyez ensuite sur la touche `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez revenir à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela prendra un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la compilation réussit, vous verrez la sortie suivante :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte outil à UTOC ou à la carte mère avec un câble de données XT30, ci-dessous se trouve le schéma de connexion de référence
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance mesurée entre CAN H et CAN L en l'absence de courant est d'environ `60Ω`. Si elle est de `120Ω`, vous devez connecter un jumper de `120Ω`. Si elle est d'environ `40Ω`, retirez le jumper
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_pro/flash/img/120.webp').default} size="80%" align="center" />
