# Compilation du firmware CAN version 072

## Commencer la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Sélectionnez **GPIO pins to set at micro-controller startup**, entrez : !PC13 Veuillez noter que c'est en majuscules
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/072.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis sur **Save configuration**, et enfin sur la touche `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez être retourné à l'interface de ligne de commande
    * Tapez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * La compilation est réussie si la sortie finale est la suivante :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Flashage du firmware

    * Avant de flasher le firmware, connectez la carte d'outil à l'UTOC ou à une carte mère avec un firmware de pont, en utilisant un câble de données XT30. Voici un schéma de câblage de référence
    * Veuillez vérifier que les commutateurs sont à la position correcte
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/sht36v2_utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L, mesurée avec un multimètre en mode résistance, est d'environ `60Ω` lorsque l'alimentation est coupée. Si elle est de `120Ω`, vous devez ajouter un cavalier de `120Ω`. Si elle est d'environ `40Ω`, vous devez retirer le cavalier
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v2/flash/img/120.webp').default} size="50%" align="center" />
