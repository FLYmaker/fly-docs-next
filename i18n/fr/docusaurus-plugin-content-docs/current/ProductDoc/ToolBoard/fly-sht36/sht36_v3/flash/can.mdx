# Compilation du firmware CAN

## Début de la compilation

    <SSH />

    <MakeOperate/>

    <CAN />

    * Choisissez les broches GPIO à définir au démarrage du microcontrôleur, entrez : !gpio5 Veuillez noter que ! est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/can.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, lorsque **Save configuration** apparaît, appuyez ensuite sur la touche `Y`
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez être retourné à l'interface de ligne de commande
    * Entrez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la compilation est réussie, le résultat final affichera :

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## Enregistrement du firmware
         
    * Avant d'enregistrer le firmware, connectez la carte outil à l'UTOC ou à la carte mère avec le firmware de pontage à l'aide d'un câble de données XT30, ci-dessous se trouve le schéma de câblage de référence
    * Veuillez vérifier que le commutateur DIP est bien positionné
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * Assurez-vous que la résistance entre CAN H et CAN L, mesurée avec un multimètre lorsque l'alimentation est coupée, est d'environ `60Ω`. Si elle est de `120Ω`, vous devez connecter un shunt de `120Ω`. Si elle est d'environ `40Ω`, vous devez retirer le shunt
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/120.webp').default} size="100%" align="center" />
