# Compilation du firmware RS232

## Commencer la compilation

    <SSH />

    <MakeOperate/>

    <RS232 />

    * Sélectionnez les GPIO pins à définir au démarrage du microcontrôleur, entrez : !gpio5 Veuillez noter que ! est en majuscule
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/rs232.webp').default} size="100%" align="center" />
    * Appuyez sur la touche `Q`, puis lorsque **Enregistrer la configuration** apparaît, appuyez sur la touche `Y`
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * La configuration devrait maintenant être sauvegardée et vous devriez être de retour à l'invite de commande
    * Entrez `make -j4` pour commencer la compilation, cela peut prendre un certain temps
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * Si la compilation est réussie, vous verrez le message suivant :

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```



## Entrer en mode flashage

    :::tip

    La flèche indique la touche `BOOT`

    :::

    * Avec la carte complètement déconnectée, maintenez la touche BOOT enfoncée, puis connectez le câble de données à la carte et à votre appareil
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/boot.webp').default} size="100%" align="center" />
