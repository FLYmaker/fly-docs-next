# ファームウェアの説明

    :::danger 注意事項

    * Klipperファームウェアは上位機ファームウェアを使用してコンパイルするのが最適です
    * 他の人がコンパイルしたものやファームウェアのバージョンが異なると、さまざまな問題が発生する可能性があります
    * 可能な限り改造された上位機を使用しないでください

    :::

## 特別な説明

    :::info 特別な説明
    * FLYのほとんどのマザーボードは出荷時に`Katapult`ファームウェアが焼かれています
    * FLYマザーボードは現在主に`Katapult`ファームウェアを使用して`Klipper`ファームウェアを書き込みます
    * DP5、D5、D7の3つのマザーボードは、対応する`Katapult`ファームウェアを手動で焼く必要があります
    * 以前のファームウェアが影響を与えるため、`Katapult`ファームウェアを焼き終えた後、マザーボードIDを検索するには、2回マザーボードをリセットする必要があります。そうしないとファームウェアの書き込みができません
    :::

## ファームウェアコンパイル説明

    :::tip

    * FLYマザーボードはKatapultとメモリカードのブートローダーを使用してファームウェアを焼きます
    * Katapultファームウェアの利点は、接触せずに直接ファームウェアを更新できることです
    * ブートローダーファームウェアの利点は、間違ったファームウェアを焼く心配がないことです
    * ファームウェアの設定には`MobaXterm`を使用するのが最適です。ユーザーのフィードバックによると、一部の特殊なSSHソフトウェアはコンパイルに失敗することがあります
    
    :::

    * どのファームウェアを使用する場合でも、ファームウェアが正しくコンパイルされていることを確認する必要があります。そうでないと、正常なプリントができなかったり、デバイスが見つからなかったりします

## ファームウェアコンパイル説明

    :::tip
    * `Enable extra low-level configuration options`このオプションを選択しないと、具体的な内容が表示されません
    :::

### STM32

    * STM32ファームウェアをコンパイルする際に`GPIO pins to set at micro-controller startup`を設定する必要がある場合、大文字で入力してください

### RP2040

    * RP2040ファームウェアをコンパイルする際に`GPIO pins to set at micro-controller startup`を設定する必要がある場合、小文字で入力してください

## ファームウェア書き込み説明

    :::danger 注意事項

    * `Katapult`ファームウェアを使用して`Klipper`ファームウェアを書き込む場合、デバイスが見つかることを確認してください。デバイスが見つからない場合は、マザーボードを2回手動でリセットする必要があります
    * チュートリアル以外の方法でファームウェアを書き込まないでください
    * ファームウェアにコンパイルエラーがないか確認してください。コンパイルエラーがあると、正常にファームウェアを書き込むことができません

    :::

### Katapultファームウェア

    * Katapultファームウェアでコンパイルエラーがあるファームウェアを書き込んだ場合の解決方法
    * マザーボードを2回リセットしてください。リセット後、通常はLEDが点滅します
    * マザーボードのファームウェアをクリアし、再度Katapultファームウェアを書き込みます

### ブートローダーファームウェア

    * ブートローダーファームウェアでコンパイルエラーがあるファームウェアを書き込んだ場合の解決方法
    * TFカードを使用して再び書き込みます
    * 書き込みできない場合は、まず電源を切り、メモリカードをマザーボードに取り付けた後、電源を入れます。5秒で書き込み可能です
