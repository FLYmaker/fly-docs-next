# ファームウェア書き込み説明

    * ERCFはCAN接続方式で上位機と接続することができます
    * CAN接続には`XT30(2+2)-F`を使用する必要があります
    * 以下に`XT30(2+2)-F`データ線のスタイルを示します
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意事項
* メインボードのファームウェア書き込みにはKatapultを使ってKlipperを書き込む必要があります
* メインボードにKatapultが既に書き込まれていると確定している場合は、最後のKatapultファームウェア書き込みは不要です。これはメインボードのKatapultファームウェアが書き込みに失敗した場合の予防措置です。
* リセットボタンを2回素早く押してメインボード上にLEDが点滅するか確認してください。点滅している場合は、メインボードがKatapultファームウェアに入っていることを示します。
:::

## どのファームウェアが必要かを判断する方法

    * ERCFは以下の方法で上位機に接続できます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア書き込み</Button>
        * メリット：Katapultファームウェアをインストールすれば、メインボードに触れることなくファームウェアを更新でき、バス接続をサポートします。
        * デメリット：`KlipperScreen`と共に使用すると、原点復帰タイムアウト等のエラーが発生する可能性があります。
        * ヒント：CANを使用する場合は、上位機がCANをサポートしているか確認し、UTOCまたはブリッジファームウェアをインストールしたメインボードと組み合わせる必要があります。
    * <Button variant="contained" disableElevation href="./usb">USBファームウェア書き込み</Button>
        * メリット：Katapultファームウェアをインストールすれば、メインボードに触れることなくファームウェアを更新できます。ただし、メモリカードまたはDFUで書き込んだ場合は従来の方法で更新する必要があります。通信は信頼性が高く、安定しています。
        * デメリット：複数のUSBデバイスがあるとデータ転送に影響を与える可能性があります。
