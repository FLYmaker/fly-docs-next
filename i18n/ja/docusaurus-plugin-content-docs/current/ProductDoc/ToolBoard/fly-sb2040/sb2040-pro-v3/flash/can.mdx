# CAN ファームウェアのコンパイル

## コンパイルの開始

    <SSH />

    <MakeOperate/>

    <CAN />

    * マイクロコントローラの起動時に設定するGPIOピンを選択し、入力：!gpio5 注意！は大文字
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/can.webp').default} size="100%" align="center" />
    * `Q`キーを押して、**Save configuration**が表示されたら、再度`Y`キーを押します
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * これで設定が保存され、コマンドラインプロンプトに戻ります
    * `make -j4`を入力してコンパイルを開始します、時間が少しかかるかもしれません
    <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 最後に以下の内容が出力されればコンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## ファームウェアの書き込み

    * ファームウェアを書き込む前に、XT30データケーブルを使用してツールボードをUTOCまたはブリッジファームウェアを書き込んだメインボードに接続します。以下は参考になる配線図です
    * ディップスイッチが正しい位置に設定されているかどうかを確認してください
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * CAN HとCAN Lが断電状態でマルチメーターで測定した場合、抵抗値が約`60Ω`であることを確認してください。`120Ω`であれば`120Ω`のディップスイッチをオンに、`40Ω`前後であればディップスイッチをオフにしてください
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/120.webp').default} size="100%" align="center" />
