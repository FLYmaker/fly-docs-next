# RS232ファームウェアのコンパイル

## コンパイルの開始

    <SSH />

    <MakeOperate/>

    <RS232 />

    * マイクロコントローラの起動時に設定するGPIOピンを選択します。入力：!gpio5 注意してください！は大文字です
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/rs232.webp').default} size="100%" align="center" />
    * `Q`キーを押すと**Save configuration**が表示されます。この時、`Y`キーを押します
    <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * 今、設定が保存され、コマンドラインインターフェースに戻ったはずです
    * `make -j4`と入力してコンパイルを開始します。時間が少しかかります
    <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 最後に以下の内容が出力されれば、コンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating uf2 file out/klipper.uf2
        ```


## 焼き込みモードへの進入

    :::tip

    左側が`RESETボタン`、右側が`BOOTボタン`です

    :::

    * DFUモードに入る方法は2つあります
    * 第一の方法：完全に電源を切った状態で、BOOTボタンを押しながらデータ転送用のケーブルを主板とホストPCに接続します
    * 第二の方法：データ転送用のケーブルを主板とホストPCに接続した状態で、BOOTボタンを押しながらRSTボタンを押し、RSTボタンを離してからBOOTボタンを離します
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sb2040/sb2040-pro-v3/flash/img/boot.webp').default} size="100%" align="center" />
