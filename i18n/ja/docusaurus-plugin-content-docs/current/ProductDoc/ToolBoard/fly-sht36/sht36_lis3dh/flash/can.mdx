# CAN ファームウェアのコンパイル

## コンパイルの開始

    <SSH />

    <MakeOperate/>

    <CAN />

    * GPIOピンの設定を選択するには、マイクロコントローラの起動時に「!gpio20」と入力します。注意：!は大文字です
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/can.webp').default} size="100%" align="center" />
    * `Q`キーを押すと **Save configuration** が表示されるので、その時に `Y`キーを押します
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.webp').default} size="50%" align="center" />
    * これで設定が保存され、コマンドラインプロンプトに戻ったはずです
    * `make -j4` を入力してコンパイルを開始します。時間がかかる場合があります
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.webp').default} size="50%" align="center" />
    * 最後に以下の内容が出力されたらコンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## ファームウェアの書き込み

    * ファームウェアの書き込み前に、XT30データケーブルを使ってツールボードをUTOCまたはブリッジファームウェアが書き込まれたメインボードに接続してください。以下は参考接続図です
    * ダイポールが正しい位置にあるかどうかを確認してください
        <ImageView image={require('@site/docs/General/get-id/img/utoc.webp').default} size="80%" align="left" />
    * 電源を切った状態でCAN HとCAN Lの間の抵抗値が`60Ω`程度になるかどうかをテスターで測定してください。`120Ω`の場合は`120Ω`のジャンパーを接続する必要があり、`40Ω`程度の場合はジャンパーを取り外す必要があります
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_lis3dh/flash/img/120.webp').default} size="50%" align="center" />
