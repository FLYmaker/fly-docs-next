# ファームウェア書き込み説明

    * SHT36 PROはCAN接続方式でホストマシンと接続できます
    * CAN接続には`XT30(2+2)-F`を使用する必要があります
    * 以下は`XT30(2+2)-F`データケーブルのスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.webp').default} size="80%" align="left" />

:::danger 注意
* メインボードのファームウェア書き込みにはKatapultを通じてKlipperを書き込む必要があります
* メインボードにKatapultが既に書き込まれていることが確かであれば、最後のKatapultファームウェアの書き込みは不要です、それはメインボードのKatapultファームウェアが成功しなかった場合の予防措置です
* 2回リセットを素早く押して、メインボード上にLEDが点滅するかどうかを確認できます、点滅する場合はメインボードがKatapultファームウェアに入ったことを示します
:::

## CANファームウェアのビルド

    * SHT36 PROはCAN方式でのみホストマシンに接続できます
    * <Button variant="contained" disableElevation href="./can">CANファームウェアの書き込み</Button>
        * 利点：`Katapult`ファームウェアを書き込むと、メインボードに触れることなくファームウェアを更新でき、さらにバス接続をサポート
        * 欠点：`KlipperScreen`と一緒に使用すると、原点復帰タイムアウト等のエラーが発生する可能性があります
        * 注意：CANを使用する場合、ホストマシンがCANをサポートしているか確認し、UTOCまたはブリッジファームウェアを書き込んだメインボードを準備する必要があります
