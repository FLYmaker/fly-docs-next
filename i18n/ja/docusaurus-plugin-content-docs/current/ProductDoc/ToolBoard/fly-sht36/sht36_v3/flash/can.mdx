# CAN ファームウェアのコンパイル

## コンパイルの開始

    <SSH />

    <MakeOperate/>

    <CAN />

    * GPIOピンをマイクロコントローラ起動時に設定する選択肢を選び、入力：!gpio5 注意：!は大文字
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/can.png').default} size="100%" align="center" />
    * `Q` キーを押し、**Save configuration**が表示されたら、`Y` キーを押します
        <ImageView image={require('@site/docs/General/dfu-flash/img/configuration.png').default} size="50%" align="center" />
    * これで設定が保存され、コマンドラインインターフェースに戻ります
    * `make -j4` を入力してコンパイルを開始します。時間がかかります
        <ImageView image={require('@site/docs/General/dfu-flash/img/make.png').default} size="50%" align="center" />
    * 最後に以下の内容が出力されれば、コンパイル成功です

        ```bash
        Linking out/klipper.elf
        Creating hex file out/klipper.bin
        ```

## ファームウェアの書き込み

    * ファームウェアを書き込む前に、XT30データケーブルを使用してツールボードをUTOCまたはブリッジファームウェアがインストールされたメインボードに接続してください。以下は参考接続図です
    * ディップスイッチが正しい位置に設定されていることを確認してください
        <ImageView image={require('@site/docs/General/get-id/img/utoc.png').default} size="80%" align="left" />
    * 電源が切れている状態でCAN HとCAN Lの間の抵抗値をマルチメータで測定し、約`60Ω`であることを確認してください。もし`120Ω`であれば、`120Ω`のジャンパーを接続する必要があります。`40Ω`程度であればジャンパーを取り外す必要があります
        <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/flash/img/120.png').default} size="100%" align="center" />
