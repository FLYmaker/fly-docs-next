# ファームウェア書き込み説明
    
    * ファームウェアのビルドと書き込みは上位機を使用する必要があります
    * SHT36 V3はRS232、CANの2種類の接続方法で上位機に接続できます
    * RS232とCANは`XT30(2+2)-F`を使用して接続する必要があります
    * 以下は`XT30(2+2)-F`データケーブルのスタイルです
        <ImageView image={require('@site/docs/General/get-id/img/xt30.png').default} size="80%" align="left" />

:::danger 注意事項
* メインボードのファームウェア書き込みにはKatapultを使ってKlipperを書き込む必要があります
* メインボードにすでにKatapultが書き込まれている場合は、最後のKatapultファームウェアの書き込みは不要です。これはメインボードのKatapultファームウェアが正常に書き込まれていない場合の予防措置です
* リセットボタンを2回素早く押して、メインボード上のLEDが点滅するかどうか確認できます。点滅する場合は、メインボードがKatapultファームウェアに入っていることを意味します
:::

## どのファームウェアが必要かを判断する方法

    * SHT36 V3は以下の方法で上位機に接続できます
    * <Button variant="contained" disableElevation href="./can">CANファームウェア</Button>
        * メリット：`Katapult`ファームウェアを刷ると、メインボードに触れることなくファームウェアを更新でき、バス接続もサポート
        * デメリット：`KlipperScreen`と一緒に使用すると、ホーム位置のタイムアウトなどのエラーが発生する可能性があります
        * 注意：CANを使用する場合は、上位機がCANをサポートしているか確認し、UTOCまたはブリッジファームウェアを刷ったメインボードが必要です
    * <Button variant="contained" disableElevation href="./rs232">RS232ファームウェア</Button>
        * メリット：特別なソフトウェアは不要で、通信が信頼性があり、安定しています
        * デメリット：ファームウェアの再書き込みには手動でDFUに入るか、2回RSTをダブルクリックして`Katapult`に入って更新する必要があります
        * 注意: G2TまたはUTORと一緒に使用する必要があります
