# 백업 설정 방법

:::danger 주의사항

* 본 튜토리얼은 FLY-Pi, 라즈베리파이, 오렌지파이 등의 상위 컴퓨터에 적용됩니다.
* EMMC를 사용하는 경우, 번인 모드로 들어갈 수 있어야 하며, 그렇지 않으면 본 튜토리얼이 적용되지 않습니다.
* 본 튜토리얼은 Windows 시스템에서의 작업을 기반으로 하며, 알 수 없는 디스크를 포맷하지 마세요!!!
* 포맷하지 마세요!!!    포맷하지 마세요!!!  포맷하지 마세요!!!

:::

## 필요한 소프트웨어 다운로드

    * 클릭 후 다운로드, 웹페이지에서 컴퓨터에 적합한 버전을 알려줍니다.
    * 소프트웨어는 그린 버전으로, 설치가 필요 없으며 다운로드 후 압축을 해제하여 사용할 수 있습니다.
    * <Button variant="contained" disableElevation href="https://www.diskgenius.cn/download.php">DG 하드 디스크 도구</Button>

## 백업 시작

    :::info
    * 포맷하지 마세요!!!
    * 포맷 후에는 복구할 수 없습니다!!!
    * TF 카드는 직접 컴퓨터에 연결하면 됩니다.
    * EMMC는 수동으로 번인 모드에 들어가야 합니다!!!
    :::

### 연결
    * 백업할 시스템을 컴퓨터에 연결합니다.
    * `DiskGenius.exe`를 엽니다.
    * 정상적으로 작동하면 백업할 디스크가 표시됩니다.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/boot.webp').default} size="80%" align="center" />
    
### 탐색
    * 왼쪽에서 주 분할을 클릭하고, 상단의 파일 탐색을 클릭하면 아래에 파일이 나타납니다.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/home.webp').default} size="80%" align="center" />
    * `home`을 찾아서 더블 클릭합니다.
    * 들어가면 일반적으로 하나의 폴더만 있으며, 이름은 시스템 사용자 이름입니다.
    * 두 개 이상이 있는 경우, 시스템 설정이 있는 폴더를 스스로 결정해야 합니다.
    * 그런 다음 더블 클릭합니다.
    * `printer_data`를 찾습니다.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/printer-data.webp').default} size="80%" align="center" />
    * `printer_data` 폴더에는 인쇄 파일, klipper 설정, klipper 로그 등이 포함되어 있으며, 전체를 직접 내보낼 수 있습니다.

### 설정 내보내기
    * `printer_data`를 오른쪽 클릭하면 아래와 같은 메시지가 나타납니다.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/export.webp').default} size="40%" align="center" />
    * 저장할 디렉토리를 선택하거나 직접 내보낼 수 있습니다.
    * 내보내기를 확정합니다.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/export1.webp').default} size="80%" align="center" />
    * 내보낸 파일을 열고 `config`를 찾아서 시스템 설정을 확인합니다.
        <ImageView image={require('@site/docs/DebugDoc/BasicTutorial/img/backup/export2.webp').default} size="80%" align="center" />
    * 설정 내보내기가 완료되었습니다.
    * `moonraker` 파일을 다른 시스템에 임의로 업로드하지 마세요, 시스템이 다를 경우 klipper 연결이 불가능할 수 있습니다.
