* 상위 컴퓨터가 정상적으로 네트워크에 연결되었는지 확인하세요
     * 하위 컴퓨터가 상위 컴퓨터와 데이터 전송 기능을 가진 데이터 케이블로 연결되었는지 확인하세요
     * SSH 연결을 통해 상위 컴퓨터에 접속한 후 `lsusb`를 입력하고 엔터를 누르면 다음 세 가지 상황이 발생할 수 있습니다
         * 인식된 USB 장비가 표시됩니다
         * `ls` 명령어가 없다는 메시지가 표시되면, 다음 명령어를 실행하여 설치할 수 있습니다
             ```bash
             sudo apt-get install usbutils
             ```
         * 오류 메시지 없이 다른 정보가 없으면 (시스템 문제로 인해 시스템을 교체해야 합니다)
     * `lsusb`를 입력한 후 반응이 없으면, 이는 시스템 문제로 인해 여기서는 해결할 수 없으며 정상적인 시스템으로 교체해야 합니다
     * 아래 이미지에서 원 안에 표시된 정보가 나타나면 다음 단계로 진행할 수 있습니다

     * `1d50:606f`는 이번에 사용할 장비에 속합니다. 후속 메시지는 신경 쓸 필요가 없습니다. 시스템 문제로 인해 완전히 표시되지 않거나 아예 표시되지 않을 수 있습니다
     * 두 개의 `1d50:606f`가 있으면, 왜 두 개가 있는지 확인하고 그 중 하나를 제거한 후 시스템을 재부팅해야 합니다
     * 두 개의 `1d50:606f`는 후속 펌웨어와의 연결에 영향을 미칠 수 있으며, 예를 들어 `FLY MINI PAD`의 경우, 다른 CAN 브리징 장비를 사용하지 않고 내장 UTOC만 사용하는 것이 좋습니다
