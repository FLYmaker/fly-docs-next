# 펌웨어 번인 설명
    
    * 펌웨어 컴파일과 펌웨어 번인 모두 상위 기기를 사용해야 합니다.
    * SHT36 V3는 RS232, CAN 두 가지 연결 방식으로 상위 기기와 연결할 수 있습니다.
    * RS232와 CAN은 `XT30(2+2)-F`를 사용하여 연결해야 합니다.
    * 아래는 `XT30(2+2)-F` 데이터 케이블 스타일입니다.
        <ImageView image={require('@site/docs/General/get-id/img/xt30.png').default} size="80%" align="left" />

:::danger 참고 사항
* 메인보드 펌웨어 번인에는 Katapult를 통해 Klipper를 번인해야 합니다.
* 만약 메인보드에 이미 Katapult가 번인되어 있음을 확실히 안다면, 마지막에 있는 Katapult 펌웨어 번인은 필요하지 않습니다. 이는 메인보드의 Katapult 펌웨어 번인이 실패했을 경우를 대비한 것입니다.
* 두 번 빠르게 리셋을 누른 후 메인보드에 LED가 깜빡이면 메인보드가 Katapult 펌웨어로 들어갔음을 나타냅니다.
:::

## 필요한 펌웨어를 어떻게 판단할까요?

    * SHT36 V3는 다음 방식으로 상위 기기와 연결할 수 있습니다.
    * <Button variant="contained" disableElevation href="./can">CAN 펌웨어</Button>
        * 장점: `Katapult` 펌웨어를 번인하면 메인보드를 만지지 않고도 펌웨어를 업데이트할 수 있으며, 버스 연결을 지원합니다.
        * 단점: `KlipperScreen`과 함께 사용할 경우 홈 위치 시간 초과 등의 오류가 발생할 수 있습니다.
        * 참고: CAN을 사용하려면 상위 기기가 CAN을 지원하는지 확인하고, UTOC 또는 브리징 펌웨어가 번인된 메인보드와 함께 사용해야 합니다.
    * <Button variant="contained" disableElevation href="./rs232">RS232 펌웨어</Button>
        * 장점: 특별한 소프트웨어가 필요 없으며, 통신이 안정적입니다.
        * 단점: 펌웨어를 다시 번인하려면 수동으로 DFU 모드에 들어가거나 RST를 두 번 누르고 `Katapult`로 들어가야 업데이트할 수 있습니다.
        * 참고: G2T 또는 UTOR와 함께 사용해야 합니다.
