# Расширенные функции KlipperScreen

## Как включить курсор мыши в KlipperScreen

    * Откройте файл `KlipperScreen.conf` в конфигурации принтера, который создается автоматически после установки KlipperScreen. Если его нет, создайте сами.
        <ImageView image={require('./img/ks/conf.png').default} size="100%" align="center" />
    * Добавьте следующий код в начало конфигурации:
    ```cfg
    [main]
    show_cursor: True
    ```
        <ImageView image={require('./img/ks/conf-1.png').default} size="80%" align="center" />
    * Сохраните и перезапустите.

## Подключение к удаленным хостам

    * Откройте файл `KlipperScreen.conf` в конфигурации принтера, который создается автоматически после установки KlipperScreen. Если его нет, создайте сами.
        <ImageView image={require('./img/ks/conf.png').default} size="100%" align="center" />
    * Добавьте следующий код в начало конфигурации:
    * Можно изменить имя принтера `VORON-300` и IP-адрес `192.168.6.217` устройства, к которому нужно подключиться.
    * Обратите внимание, что второй `[printer KlipperScreen]` нужен для подключения к удаленному устройству для отладки или выключения.
    :::danger Важно

    * Убедитесь, что вы находитесь в одной локальной сети.

    :::

    ```cfg
    # Определение принтера и его имени. Имя - это все, что после слова printer
    [printer VORON-300]
    # Определение хоста и порта Moonraker, если они отличаются от 127.0.0.1 и 7125
    moonraker_host: 192.168.6.217
    # порты 443 и 7130 будут использовать https/wss
    moonraker_port: 7125
    # Ключ API Moonraker, если подключение не с доверенного IP-адреса клиента
    moonraker_api_key: False

    # Определение принтера и его имени. Имя - это все, что после слова printer
    [printer KlipperScreen]
    # Определение хоста и порта Moonraker, если они отличаются от 127.0.0.1 и 7125
    moonraker_host: 127.0.0.1
    # порты 443 и 7130 будут использовать https/wss
    moonraker_port: 7125
    # Ключ API Moonraker, если подключение не с доверенного IP-адреса клиента
    moonraker_api_key: False

    ```
