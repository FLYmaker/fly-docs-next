# CAN мост

CAN шина - это последовательный протокол связи, используемый для реального времени, который может передавать сигналы по витой паре, являясь одним из самых широко применяемых в мире полевых шин. Протокол CAN используется для общения между различными компонентами в автомобиле, заменяя дорогостоящие и громоздкие жгуты проводов. Его надежность расширяет применение на другие автоматизированные и промышленные задачи. Характеристики CAN протокола включают целостность передачи данных, поддержку в реальном времени, скорость передачи до 1Mb/s, 11-битную адресацию и возможность обнаружения ошибок.
В 3D принтерах CAN шина используется для уменьшения количества проводов, соединяющих печатающую головку с основной платой, что раньше требовало нескольких десятков проводов, а теперь только четыре, значительно упрощая прокладку проводов.
В этой главе будет дан краткий обзор использования моста CAN.

## Использование CAN для подключения инструментальной платы к основной плате

    * Обратите внимание, что питание инструментальной платы подключается не к порту CAN, а к линии питания 12V-24V.
    <ImageView image={require('@site/docs/ProductDoc/MainBoard/fly-gemini/fly-gemini-v3/img/canbridge.png').default} size="80%" align="center" />
    * Обратите внимание, что 120Ω перемычка должна быть установлена.
    <ImageView image={require('@site/docs/ProductDoc/MainBoard/fly-gemini/fly-gemini-v3/img/120.png').default} size="80%" align="center" />

## Обновление прошивки основной платы

* Убедитесь, что прошивка на плате Gemini настроена как `USB мост CAN`.
