# Использование Eddy

    :::info

    * Обратите внимание, рекомендуется использовать отдельный Z-концевик, не используйте сканирование платформы в качестве Z-концевика
    * При установке Eddy, основание должно быть на 1-2 мм выше сопла
    * При калибровке частоты Eddy, катушка PCB должна быть на расстоянии не менее 5 мм от нагревательной платформы
    * Пожалуйста, не проводите тесты на холодной платформе и не включайте нагревательную платформу во время тестирования

    :::

## Справочная конфигурация и инструкции по использованию

### Изменение частоты по умолчанию для ldc16121612
    Этот метод применим только для sht36-v3/SB2040-v3 или SB2040-Pro-v3
    Для выполнения команд можно использовать инструмент Mobaxterm

    Адрес для скачивания:https://mobaxterm.mobatek.net/download.html
    
    [Подробная инструкция](/docs/ProductDoc/SBC/fly-pi-v2/ssh)


    Выполните первую команду и нажмите Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Выполните вторую команду и нажмите Enter
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    Если замена прошла успешно, вернется строка с `LDC1612_FREQ = 40000000`, что означает, что замена прошла успешно. Если ничего не вернулось, то замена не удалась.
    Или используйте эту команду для проверки успешности
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
sed -n '10,15p' ~/klipper/klippy/extras/ldc1612.py
```

   Важное замечание: После изменения используйте эту команду для перезапуска службы klipper и повторного выполнения калибровки частоты Eddy
   ```cfg
   systemctl restart klipper
   ```
   Команда калибровки частоты Eddy
   ```cfg
   PROBE_EDDY_CURRENT_CALIBRATE CHIP=fly_eddy_probe
   ```

    ### Конфигурация сканирования платформы

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 0.8
        i2c_address: 43
        i2c_mcu: SB2040
        i2c_bus: i2c1b
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Обратите внимание, что на данный момент нет внешнего модуля 1612, эта конфигурация временно не может быть использована
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 0.8
        i2c_address: 42
        i2c_mcu: SB2040
        i2c_bus: i2c1b
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Конфигурация температурной компенсации

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SB2040:gpio28
        ```

### Конфигурация bed_mesh

    * Параметр `horizontal_move_z` в [bed_mesh], рекомендуется установить на 2, чтобы Eddy находился как можно ближе к платформе во время сканирования

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
