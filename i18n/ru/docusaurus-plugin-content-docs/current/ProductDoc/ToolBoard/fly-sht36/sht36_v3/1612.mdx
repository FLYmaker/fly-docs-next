# Использование Eddy

    :::info

    * Обратите внимание, что для сканирования платформы рекомендуется использовать отдельный Z-концевик, не рекомендуется использовать сканирование платформы как Z-концевик
    * При установке Eddy, его нижняя часть должна быть на 1-2 мм выше сопла
    * При калибровке частоты Eddy, катушка PCB должна быть на расстоянии не менее 5 мм от нагревательной платформы
    * Пожалуйста, убедитесь, что вся нагревательная платформа находится при комнатной температуре при тестировании, и не включайте нагревательную платформу во время тестирования

    :::

## Справочная конфигурация и инструкции по использованию

    ### Изменение частоты ldc1612 по умолчанию
    Этот метод применим только для sht36-v3/SB2040-v3 или SB2040-Pro-v3
    Можно использовать инструмент Mobaxterm для выполнения команд

    Адрес для скачивания:https://mobaxterm.mobatek.net/download.html

    [Для детального руководства обратитесь к](/docs/ProductDoc/SBC/fly-pi-v2/ssh)

    Выполните первую команду и нажмите Enter

    ```cfg
sed -i 's/LDC1612_FREQ = 12000000/LDC1612_FREQ = 40000000/g' ~/klipper/klippy/extras/ldc1612.py
    ```

    Выполните вторую команду и нажмите Enter
  ```cfg
grep "LDC1612_FREQ = 40000000" ~/klipper/klippy/extras/ldc1612.py
  ```
    Если замена прошла успешно, вернется строка с `LDC1612_FREQ = 40000000`. Если нет возврата, значит замена не удалась.
    Или используйте эту команду для проверки успешности:
    <ImageView image={require('@site/docs/ProductDoc/ToolBoard/fly-sht36/sht36_v3/img/1612.webp').default} size="110%" align="left" />
```cfg
cat /home/fly/klipper/klippy/extras/ldc1612.py
```

    ### Настройка сканирования платформы

        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 0.8
        i2c_address: 43
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

        * Обратите внимание, что в настоящее время нет внешнего модуля 1612, эта конфигурация пока не может быть использована
        ```cfg
        [probe_eddy_current fly_eddy_probe]
        sensor_type: ldc1612
        z_offset: 0.8
        i2c_address: 42
        i2c_mcu: SHT36
        i2c_bus: i2c1e
        x_offset: 0
        y_offset: 0
        speed:40
        lift_speed: 5
        ```

    ### Настройка температурной компенсации

        ```cfg
        [temperature_probe fly_eddy_probe]
        sensor_type:Generic 3950
        sensor_pin:SHT36:gpio28
        ```

### Настройка bed_mesh

    * В [bed_mesh] параметр `horizontal_move_z`, рекомендуется установить на 2, чтобы Eddy был как можно ближе к платформе при сканировании

    ```cfg
    [bed_mesh]
    horizontal_move_z: 2
    ```
